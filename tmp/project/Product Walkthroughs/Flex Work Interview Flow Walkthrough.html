01 · Setup · turn on the flag

Flip the interviews feature flag.

Every interview surface across admin, manager, agency, and worker is gated behind one flag. Off by default; turning it on unlocks the whole workflow without an app reload.

⚑
Step 01

Turn on Interviews

Lives in the standard Feature Flags surface. Once on, the four downstream sections in this walkthrough all become reachable.

Then jump to §02 →
  1. Side nav → Settings

    Open Settings.

    From any page, open the global side nav (the panel icon top-left) and click Settings. The Settings dock will open in the side nav.

  2. Settings dock → Feature Flags

    Click Feature Flags in the Settings dock.

    The Feature Flags page lists every pre-release capability tenant-wide. Each carries a summary and a per-tenant toggle.

  3. Feature Flags → Interviews

    Find “Interviews” and turn it on.

    Scroll to the Interviews entry and flip the toggle. No reload required — every consumer subscribes to the flag change and re-renders.

    What changes immediately: A new Interview tab appears in the Settings dock, an Interview pipeline pill appears in the Requisitions omnibar, and the worker mobile More screen gains an Interviews tile.
02 · Admin · configure once

Pick the workflow and configure the catalog.

A
Role

Tenant admin · program owner

Picks the tenant-wide candidate workflow (Automatic or Interview), configures the catalog, owns notification templates, signs off on the SLA policy. The Candidate workflow tab lives inside Settings → Configuration so it sits next to every other org-wide default.

Next: Manager →
Candidate workflow Type catalog Scorecards Notification templates Decline reasons Calendar integration Policies Skip approvals SLA thresholds Version history
  1. Settings dock → Configuration

    Open Settings → Configuration.

    Configuration shows two tabs in its header (Settings and Master data) and a stack of config cards on the Settings tab. With the interviews flag on, a new Candidate workflow card appears in the stack between Engagement types and Supplier types, with a matching entry in the right-rail navigation.

  2. Configuration → Candidate workflow card

    Jump to the Candidate workflow card.

    Click Candidate workflow in the right rail (or scroll to it). Two radio-style tiles appear:

    • Automatic — auto-accept the priority match and fire the offer. No interview gate. This is the default for every new tenant.
    • Interview — pause at the interview stage. Unlocks scheduling, scorecards, and the candidate portal.
    The choice persists per organization at flexwork.candidateWorkflow.{orgId}. Per-requisition overrides remain available from the Distribution edit panel.

  3. Switch to Interview to surface the admin configuration.

    Click the Interview tile. The empty “Automatic workflow is active” message inside the card is replaced with the full admin configuration surface — a left rail of nine sections with the type catalog selected by default. Everything renders inline inside the same card, below the Automatic / Interview tile pair.

  4. Tour the type catalog.

    Six seeded types — Phone screen, Video screen, Technical, Panel, On-site, Executive. Each row carries default duration, method, location pattern, the default panel composition, and the bound scorecard. Toggle Active to retire a type without deleting it.

    • Phone screen is the only type without a required-to-advance scorecard.
    • Executive ships off by default — turn it on with the Active switch.

  5. Pick “Scorecard templates” in the left rail.

    Five scorecard templates, each bound to a type. Click any template to see its competency list. The Required to advance switch in the top-right of the detail pane gates whether a candidate can be moved out of the Interview stage without that scorecard submitted.

    Try flipping “Phone screen” on. Then later, when the manager tries to advance Helena Voss without one, the action will fail with an audit-logged reason.

  6. Open “Notification templates”.

    Seven seeded templates — invite (candidate), invite (supplier), reminder, reschedule, cancel, outcome-advanced, outcome-declined. Click any template to edit its subject and body inline. Token slots use the {{token_name}} grammar.

  7. Visit “Decline reasons”.

    Nine seeded codes. The taxonomy is required on every backward stage transition (advance → decline / withdraw / on-hold). Toggle any code Active to retire it without losing audit history.

  8. Open “Calendar integration”.

    Three providers — Microsoft 365, Google Workspace, .ics fallback. The .ics row is always-on. Click Connect on Google to flip the demo state to connected; the conflict-check in the manager's schedule panel reacts in real time.

  9. Skim the remaining tabs.

    Interview policy pre-sets the required-toggle by supplier type and job family. Skip-interview approvals shows the queue of override requests with approve / deny actions. SLA thresholds sets the three program targets (time-to-schedule, time-to-feedback, time-to-decision). Version history records every catalog edit.

    The admin surface persists in memory only — reloading the page returns it to the seed state. Useful for repeat demos; not yet wired to a backend.
03 · Manager · schedule + score

Run the interview from inside Requisitions.

M
Role

Hiring manager · buyer-side

Raises requisitions, reviews submittals, runs the interview, signs off the offer. Everything below sits inside the Requisitions hub — one extra pill in the omnibar swaps the table for the manager's interview surfaces.

Next: Agency →
Schedule + reschedule Multi-round graph Side-by-side compare Availability windows Mobile workflow
  1. Side nav → Requisitions

    Open Requisitions.

    Standard requisitions list. With the interviews flag on, an Interview pipeline pill appears in the omnibar between Create and the More menu.

  2. Omnibar → Interview pipeline

    Click Interview pipeline.

    The requisitions table is replaced with the manager's interview workspace. A left rail of five sections, and the submittal list selected by default. Five seeded candidates across three requisitions, each showing stage, round, score, and SLA badge.

  3. Open Helena Voss's submittal.

    Click Open on the first row. A wide side panel slides in showing the full interview history — three rounds, with the most recent scheduled for May 28. Roll-up score and outstanding scorecard counts at the top.

  4. Submit the missing scorecard inline.

    R. Devarajan's scorecard on the May 18 Technical interview is pending. Click any of the 2 / 3 / 4 / 5 rating buttons to submit on R. Devarajan's behalf. The pill flips to Submitted; the roll-up score recalculates immediately.

  5. Schedule the next round.

    Click + Schedule round 4 at the bottom of the panel. A second side panel opens with the schedule form:

    • Interview type — pre-fills duration, method, location, panel from the A1 catalog.
    • Date / time / method / location — editable.
    • Panel picker — default panel chips at the top; type a name in the search box to add interviewers from the org tree.
    • Calendar conflict check — one row per panelist with a green Free or red Busy pill, sourced from the M365 free-busy connection. The T. Donin row turns red when the time is set to 14:00 (a seeded conflict).

    Click Send invite. The new interview appears in the history with status Scheduled and a Scheduled badge.

  6. Reschedule or cancel an existing interview.

    Any Scheduled or Awaiting-confirmation interview shows Reschedule and Cancel below the status pill. Both require a reason; Cancel pulls from the A4 decline taxonomy.

  7. Switch sections in the left rail.

    Click through:

    • Multi-round graph — per-submittal progress bars with the current round and total rounds expected.
    • Side-by-side compare — three Interview-stage candidates stacked column-wise with score, source, supplier, outstanding cards.
    • Availability windows — two seeded windows (10-day and 5-day) with claimable slots in a calendar grid.
    • Mobile workflow — a phone-shaped preview of the manager-mobile scorecard submission flow.

  8. Return to the standard table.

    Click Interview pipeline in the omnibar again to toggle back. The requisitions table reappears unchanged — the interview workspace is a view swap, not a route change.

04 · Agency · respond to invites

Accept, decline, or propose alternates from the supplier inbox.

S
Role

Agency recruiter · supplier-side

Lives in Requisitions all day. Submits candidates, watches for buyer interview invites, forwards the consolidated details to the candidate. The same pill the manager uses re-labels to Interview inbox in the agency view.

Next: Worker →
Required-pill Invite inbox Accept / Propose / Decline Forward to candidate Cross-buyer pipeline Reschedule + withdraw Bulk status Funnel analytics Per-buyer SLA
  1. Avatar menu → role → Agency

    Switch the “viewing as” role to Agency.

    Click the avatar chip in the top-right of the app nav, hover the current role (Admin / Manager / MSP), and select Agency. The page reloads its IA: Suppliers is hidden, the role badge flips, and the app surfaces shift to the supplier-side view.

  2. Side nav → Requisitions

    Open Requisitions in the agency view.

    Same page as before, but the omnibar pill is now labelled Interview inbox. The list itself shows distributed reqs from the buyer's perspective.

  3. Omnibar → Interview inbox

    Click Interview inbox.

    The table swaps for the agency workspace. Left rail of seven sections; Invite inbox + actions selected by default. Every scheduled or awaiting-confirmation interview across every buyer the supplier serves appears as a card with three actions plus a Forward link.

  4. Accept an invite on a candidate's behalf.

    Click Accept on Helena Voss's May 28 panel invite. The status flips to Done; the card moves to the bottom of the list.

  5. Propose an alternate slot.

    On a different invite, click Propose alternate. A side panel opens with the original slot, a suggested-slot input, and a reason field. Submitting sends it back to the buyer manager for approval (the candidate's interview goes back to Awaiting-confirmation).

  6. Forward the details to the candidate.

    Click Forward to candidate. A panel opens with the pre-filled email subject and body (sourced from the A3 candidate-facing template). A delivery-tracking toggle defaults on; flagging the candidate if they haven't opened the email within 24 hours.

  7. Switch to the other agency sections.

    Click through the left rail:

    • Required-pill on req cards — demonstrates how the interview-required flag surfaces on every supplier-facing req card.
    • Cross-buyer pipeline — flat list of every submittal in Interview stage with status, score, last-interview date.
    • Reschedule + withdraw — per-candidate actions with reason capture.
    • Bulk status check — multi-select for the morning roll-up call.
    • Funnel analytics — 90-day submit-to-hire funnel with per-stage conversion rates.
    • Per-buyer SLA — supplier-side scorecard of each buyer's time-to-schedule, time-to-feedback, time-to-decision.
05 · Worker · candidate portal

Confirm, pick a slot, prep for the interview.

W
Role

Worker · candidate

The person being interviewed. Lands on the portal from a magic-link email — no login required. In the prototype, the portal opens from the Worker mobile preview's More menu.

Then: Teardown →
Magic-link portal Confirm / decline Prep + panel bios Self-serve slot pick Reschedule Outcome history Add to calendar In-app messaging Post-interview survey
  1. Avatar menu → Worker

    Open the Worker mobile preview.

    Switch the role back to Admin or Manager first (the agency role doesn't expose the Worker preview). Click the avatar chip, hover the role, click Worker. A phone-shaped panel docks on the right of the screen with the Worker mobile app inside.

  2. Worker mobile → More tab

    Tap the More tab in the bottom nav.

    The More screen shows a tile grid — Interviews, Availability, Skills, Certifications, Benefits, Pay stubs, Settings, Refer a friend, Help, Feedback. The Interviews tile only appears when the flag is on.

  3. More → Interviews

    Tap Interviews.

    The candidate portal opens as a stacked screen with a header rail at the top. Greets the candidate by first name, shows the upcoming interview's date / type / method / location, the panel bios, and the prep notes.

  4. Confirm attendance.

    Tap Confirm attendance. The header banner flips green; a confirmation note appears below the panel section. The decline reason picker disappears.

  5. Or request a reschedule.

    Before confirming, tap Request reschedule. A wide side panel opens with a calendar grid of self-serve slots over the next five working days. Pick a slot; tap Confirm new slot. The header re-renders with the new time.

  6. Skim the rest of the portal.

    Scroll down through the remaining sections:

    • Add to your calendar — three buttons for Outlook .ics, Google Calendar, and Apple Calendar .ics.
    • Messages with the recruiter — tap Open thread to see a sample comment thread.
    • After your interview · share feedback — an NPS-style 0–10 rating and a freeform comment, only visible after the candidate has confirmed.
    • After the interview — outcome notification history (sample data shows the previous two rounds advanced).

  7. Close the worker preview.

    Tap the back arrow in the panel header to return to More, then either tap More items in the bottom nav to keep exploring, or use the avatar menu in the main app to switch the “viewing as” role back. The worker docked panel stays open until dismissed.

06 · Teardown · turn it back off

Restore the flag-off baseline.

⛌
Reset

Turn Interviews off again

Confirms the flag-off contract: with the flag off, every interview surface disappears and the rest of the app reads byte-identical to its pre-feature shape.

Back to top ↑
  1. Settings → Configuration → Candidate workflow

    Flip the toggle off (or set workflow back to Automatic).

    Same flag, same surface. Toggle off — or alternatively leave the flag on but flip the workflow choice back to Automatic. The manager / agency / worker entry points stay accessible but every requisition skips the interview gate.

  2. Walk the app and confirm everything hid.

    • Settings dock — Settings → Configuration still shows the same two top-level tabs (Settings / Master data); the Candidate workflow card inside the Settings tab is gone.
    • Requisitions omnibar — the Interview pipeline / Interview inbox pill is gone.
    • Worker mobile → More — the Interviews tile is gone.
    • Side nav — no IA changes either way (interviews never had a top-level entry).
    What persists across the toggle: the in-memory configuration (type catalog, scorecards, templates) stays seeded. Re-enabling the flag returns the admin surface to wherever you left it.

  3. Optional · reload to reset all demo state.

    The in-memory store ignores the flag flip but reloading the page returns the catalog, scorecards, templates, policies, and submittal interview history to their seed values. Useful when running back-to-back demos.

What good looks like

A clean run of this walkthrough — flag on, four roles, flag off — takes about ten minutes and exercises every one of the forty surfaces in Interview Workflow Tasks. If anything looks broken or doesn't match the steps above, log it in the Flex Work v2 changelog with a cl-tag--fix entry pointing at the offending surface.
