What this is. A click-by-click walkthrough of the full worker on-/offboarding flow inside Flex Work — Admin configures the two lifecycle programs, Manager onboards a Professional and a direct-sourced Frontline worker side by side, the workers self-serve through their mobile shells, and Manager closes the engagement when it ends. Every surface below ships in the live prototype.

Time to run. ~12 minutes end-to-end.

Companion. On-/Offboarding parity tasks — the task list with every surface mapped to its host file.

Flex Work VMS · On-/Offboarding Flow Walkthrough · v1.0

See the entire worker lifecycle in twelve minutes.

Nine sections, three roles. Flip one flag, configure two program cards, onboard a Professional consultant under the Fieldglass + VNDLY shape and a Frontline shift worker under the Joined Up shape — all through the same panel host. Follow each worker through the mobile shell as they sign documents, complete RTW, and acknowledge the handbook. Track progress on the worker detail, then walk the same workers out the other end through a scoped end-engagement panel that closes assets, revokes access, and reconciles final pay. Same code path everywhere; zero new top-level pages.

01 · Setup · turn on the lifecycle

One existing flag for Pro, one new flag for Frontline.

The Pro lifecycle rides on the existing professionalWork derivation — turn on any Professional engagement type in Settings → Configuration → Engagement types and it lights up. Frontline direct-sourcing sits behind a new flag, frontlineDirect, default off so the existing demo (which doesn't direct-source Frontline) stays byte-identical until you opt in. Both flags broadcast on featureflags:change so every surface re-renders without a reload.

⚑
Step 01

Enable the two lifecycle programs

One toggle each. No app reload required.

Next: Admin →
  1. Side nav → Settings → Configuration → Engagement types

    Open the Engagement types card.

    The card lists the four canonical engagement types — Shift, Assignment, Project, Statement of Work. Any of Assignment / Project / SOW being on derives professionalWork = true; that's what gates the Pro lifecycle card on the next section.

    If you're on Helios Power Generation: all three Pro engagement types ship on by default. Skip to step 03.
  2. Side nav → Settings → Feature flags → Misc

    Flip Frontline direct sourcing to on.

    The new Frontline direct sourcing flag sits in the Misc group, default off. Flipping it unlocks three things at once: the Frontline lifecycle program card in Settings, the worker-type variant on the Add panel, and the pool-conditional Onboarding tab on worker mobile. Agency-sourced Frontline workers stay untouched — their lifecycle still belongs to the agency.

    Flag-off contract: turn frontlineDirect back off and the Frontline lifecycle card disappears, the Add panel hides the worker-type segmented control, and worker mobile drops the On-board tab. The existing prototype renders identically to today.

  3. Confirm the surfaces light up.

    Three things change immediately:

    • Settings → Configuration grows two new rail items — Pro lifecycle and Frontline lifecycle — alongside the existing SOW program + IC program.
    • The Workforce omnibar Create dropdown gains an Invite Frontline worker action next to Add Professional.
    • Worker detail pages on direct-sourced workers (Internal / Float / Per-diem / Alumni / Professional) gain a new accordion: Onboarding when their status is Onboarding, Offboarding when Offboarding.
02 · Admin · configure the programs

Author the task catalogs, connectors, and approval thresholds — once, for every downstream surface.

A
Role

Tenant admin · program owner

Owns both lifecycle programs. The task catalogs authored here are read by the Add panel (step-3 task set), the OnboardingTracker accordion on every worker detail, and the worker mobile Onboarding tab — one source of truth, three surfaces.

Next: Manager →
Pro lifecycle · Onboarding Pro lifecycle · Offboarding Pro lifecycle · Approval Pro lifecycle · Connectors Frontline lifecycle · same 4 tabs
  1. Settings → Configuration → Pro lifecycle

    Scroll to the Professional lifecycle card.

    The card sits between IC program and Program (funding) in the Configuration stack. Four sub-tabs: Onboarding · Offboarding · Approval · Connectors. Card head carries the Restore defaults action.

    Mirrors the same composite-card pattern the SOW program and IC program cards use. Same shape, different catalog — the only thing that's new is the task data.

  2. Tour the Onboarding tab.

    Eleven Fieldglass + VNDLY-shaped tasks seeded by default — MSA, SOW, Mutual NDA, IP assignment, Background check, I-9 / Right to Work, Tax form, Direct deposit, Hardware provisioning, System access (IdP, email), Orientation. Each task row shows owner (Worker / Shared / Employer), connector chip (E-Sig / BG check / RTW / Payroll / Banking / SSO / Asset / Manual), required flag, and an editable "Due in N days" stepper.

    Try: Disable Hardware provisioning, then jump to step 12 (the Add panel) — it won't appear in the worker's task list. Re-enable it to restore.

  3. Switch to the Offboarding tab.

    Six closeout tasks — Asset return, Revoke system access, Knowledge transfer, Exit interview, Final-pay reconciliation, Do-not-rehire review. Same row shape as Onboarding; default due-days lean longer (5–14d) because closeout has more lead time than launch.

  4. Visit the Approval tab.

    Four policy fields: BG-check risk threshold for Compliance review (25 / 35 / 50 / 65), Onboarding SLA in days, Severance threshold for HR-Legal review (10K / 25K / 50K / 100K), and Offboarding SLA. Below: a single policy toggle — Do-not-rehire flag requires HR-Legal sign-off, on by default.

    Same router primitive the SOW + IC programs already use. The thresholds set here gate the panels the Manager will walk through in sections 03 and 07.

  5. Open the Connectors tab.

    Seven connector rows — Background check (Checkr · Sterling · HireRight), Right to Work (E-Verify · Sterling RTW · Trust ID), E-Sig (DocuSign · Adobe Sign, reuses SOW config), SSO provisioning (Okta SCIM · Azure AD · Google Workspace), Asset provisioning (JAMF · Manual), Payroll handoff (Dayforce Payroll, reuses IC config), Banking validation (Plaid · Micro-deposit). Each row is an enable toggle + a provider segmented control.

    Reuse pills on E-Sig and Payroll show that the same connector configured for SOW (e-sign) or IC (payroll handoff) is the one this lifecycle uses — change it once on the canonical config card and every downstream program reads the new provider.
  6. Settings → Configuration → Frontline lifecycle

    Scroll down to the Frontline lifecycle card.

    Sits directly below the Pro lifecycle card. Same four-tab shell — Onboarding (nine Joined Up–shaped tasks: invite code, ID verification, I-9 / RTW, tax form, direct deposit, handbook ack, orientation video, optional BG check, shift-ready gate), Offboarding (five tasks: last-shift handling, asset return, deactivate mobile access, final-pay note, do-not-rehire flag), Approval (same policy fields, BG threshold defaults to 50 — Frontline roles are more permissive), Connectors (five rows — RTW + BG + Handbook e-Sig + Banking + Payroll, narrower than Pro because direct Frontline doesn't need IP / NDA / SSO).

    Pro vs Frontline · what differs

    Pro carries MSA + SOW + NDA + IP + SSO + Asset because Professionals get system access and sign IP. Frontline workers don't — their nine-task set is the minimum to take a shift.

    Joined Up shape

    The Frontline catalog mirrors Joined Up's "ready for shifts in days, not weeks" promise — invite code on arrival, complete RTW + tax + banking on the worker phone, watch a 5-minute orientation video, and the shift-ready gate auto-flips green.

03 · Manager · onboard a Professional

Same Add panel that ships today — pre-filled from the accepted offer.

M
Role

Buyer-side hiring manager

Opens the Add Professional panel from the omnibar (or directly from the accepted offer on the Interviews surface), confirms the engagement details, picks the task set, sends the invite. Worker takes it from there on their phone.

Next: Frontline →
  1. Workforce → omnibar Create → Add Professional

    Open the Add Professional panel.

    720px side panel with a four-step wizard — Engagement → Compensation → Onboarding → Review. With frontlineDirect on, a segmented control at the top of the panel offers Professional (default) or Frontline (direct). Leave it on Professional for this section.

    The Pro variant is the byte-identical existing flow — every existing demo path still works. We're just borrowing the same shell for the Frontline shape next.

  2. Fill the Engagement step.

    Eight fields on a two-column grid: Full name, Email, Country of work (drives tax + comp rules — pick US to default the tax form to W-4), Role, Hiring manager, Source (Direct or one of three pre-configured suppliers), Term start, and a read-only Engagement type · Permanent · no end date footer.

  3. Author Compensation.

    Pick a cadence (Weekly / Monthly / Annual), enter a rate amount, choose currency. The panel auto-calculates the estimated annual value. Optional: pick Timesheet mode (Not required for deliverable-based engagements, or Required for effort logging) and the SOW template (four pre-configured shapes).

  4. Tour the Onboarding task list.

    The eleven tasks the admin authored in section 02 show up here as a togglable list, defaults reflecting the Required flag. Disable tasks that don't apply (Hardware provisioning rarely applies to an offshore consultant; Benefits enrollment only for eligible engagements).

    One catalog, three surfaces: the task list rendered here is the same one the worker will see on their mobile shell in section 05, and the same one the OnboardingTracker accordion in section 06 will track progress against.

  5. Review and send the invite.

    Three review cards summarize the panel's state — Engagement, Compensation, Onboarding tasks. Tap Send invite; the worker is created with status Onboarding, the email goes out, and the audit log records the actor + timestamp.

04 · Manager · onboard a Frontline worker

Same panel, Frontline variant — Joined Up parity.

The flag-on segmented control at the top of the Add panel is the only architectural change. Switch the toggle, and every step in the wizard branches to the Frontline shape — invite method picker, hourly + classification compensation, the nine-task Joined Up catalog. Same Interactions bus, same panel host, same four-step shell.

M
Role

Buyer-side staffing manager

Brings a direct-sourced Frontline worker on board — Internal / Float / Per-diem / Alumni pools, never agency-sourced. The agency-sourced Frontline lifecycle stays with the agency.

Next: Worker →
  1. Workforce → omnibar Create → Invite Frontline worker

    Open the panel in Frontline mode.

    The omnibar action emits openAddFrontline() on the Interactions bus — the same bus the Add Professional action uses, with the worker-type pre-set to Frontline (direct). Panel title flips to Invite Frontline worker, subtitle reads Direct-sourced · Joined Up parity.

  2. Pick an invite method.

    A new field at the top of step 1 — four options on a radio row: Email link · SMS link · 6-digit code · QR code. Picking SMS / Code / QR swaps the second field on the grid from Email to Phone so the contact field matches the channel.

    The 6-digit code and QR code paths are the Joined Up signatures — print the QR at the site, the worker scans on arrival, the app opens with the code pre-filled, and they're walking through the rest of onboarding in minutes.

  3. Fill in the worker's details.

    Field labels change for the Frontline shape: Job / role instead of Role, Site manager instead of Hiring manager, Pool (Internal / Float pool / Per-diem / Alumni) replaces the Source supplier picker, Start date replaces Term start. The Engagement type footer reads Shift · hourly.

  4. Author Compensation — hourly + classification.

    Step 2 swaps the cadence picker for a Classification radio (W-2 hourly · W-2 salaried · W-2 tipped — each row tagged with overtime eligibility and time-capture mode). Enter an Hourly rate and currency; the panel calculates an estimated weekly value (40h baseline). The timesheet field is replaced with a read-only Time capture · Clock in / out.

  5. Confirm the Joined Up task set.

    Step 3 reads its catalog from window.getOnboardingTasks("frontline") instead of the Pro hardcoded list — the nine tasks the admin authored in section 02. Toggle off anything that doesn't apply (BG check is optional and disabled for non-driver / non-cash roles by default).

  6. Send the invite.

    Review screen mirrors the Pro shape with Frontline labels. Tap Send invite; the worker enters the roster with pool: Internal (or whichever you picked) and status Onboarding. Their first app-open will be section 05.

    Same shell, two shapes: the Add panel never forked into a parallel "Add Frontline" page. The same host renders both flows; the variant flag flips ~30 fields, the rest is reused verbatim.
05 · Worker · self-serve onboarding

A new tab on the mobile shell, surfaced pool- and status-conditional.

Worker mobile already ships a four-tab shell — Home / Shifts / More plus the IC portal variant. Direct-sourced workers (Pro + Frontline direct) whose status is Onboarding or Offboarding see a new tab prepended to the bar — On-board or Off-board — with a badge counting open tasks. Agency-sourced Frontline workers never see this tab; their onboarding belongs to the agency.

W
Role

Worker · mobile-first

Sees the tasks they own — sign documents, complete RTW + tax form, add direct deposit, watch the orientation video, acknowledge the handbook. Employer tasks (asset provisioning, SSO setup) are hidden — the manager sees those on the tracker accordion.

Next: Manager tracker →
  1. Top-right user menu → View as → pick a worker in Onboarding

    Switch into worker mode.

    The mobile-frame shell takes over the viewport. Pick any direct-sourced worker — Brianna Boone (Internal), Keiko Ozawa (Float pool nurse, on Mercy industry pack), or any Pro consultant you onboarded in section 03.

    Visibility gate: if you picked an Agency-sourced Frontline worker by mistake, the On-board tab simply isn't there. That worker's onboarding lives with their agency, not us.

  2. Tap the On-board tab.

    First tab on the bottom bar with a badge showing the open task count. Tap it and the screen opens with a tonal hero block — eyebrow reads Professional engagement or Joined Up direct sourcing depending on the worker's kind, headline greets them by first name, the progress bar shows completion percent, and the meta row shows X of Y complete · NN%.

  3. Scroll the task list.

    Tasks render as full-width cards with a colored circular icon on the left (green check for done, blue lightning for in-review, red alert for blocked, gray arrow for pending), the task name, the description + "Due in Nd", and a CTA on the right (Start / In review / Action needed / Done).

    📄E-sign documents
    NDA, IP assignment, handbook acknowledgement — each opens an inline DocuSign envelope. Signed PDFs land on the worker record and the audit log.
    🪪I-9 / Right to Work
    Country-aware — US runs E-Verify, UK runs the Sterling RTW Share Code, EU runs eIDAS. Worker uploads photo ID + live selfie; the connector handles the verification call.
    💼Tax form
    W-4 (US) · P45 (UK) · TD1 (CA). Defaults to the country picked at invite; switchable if the worker lives in a different jurisdiction.
    🏦Direct deposit
    Single bank for Frontline (Joined Up shape, Plaid validated). Multi-split allowed for Pro (Fieldglass behavior).
    📺Orientation video
    Frontline-only · 5 minutes covering role + site safety + harassment policy. Completion tracked automatically.
    ✅Shift-ready gate
    Auto-checks. Closes the moment every required task above it completes — that's when the worker can start picking up shifts.

  4. Complete a task.

    Tap any task card — a focused drill opens (in production; toast in the prototype). Submission flips the task to Done, the hero progress bar advances, the badge count on the tab decrements. When every required task is complete the hero copy swaps to You're all set, <first name>. with a CTA pointing back to Home where shifts now appear.

    Joined Up promise kept: a Frontline worker with status Onboarding can hit the shift-ready gate in a single session — invite code → RTW → tax form → banking → orientation video → done. Status flips to Compliant, the On-board tab disappears, the Shifts tab takes over.
06 · Manager · track progress

One accordion on the worker detail — same catalog the worker sees, plus employer tasks.

M
Role

Manager · watching the cohort

Watches every worker in onboarding from a single accordion on the worker detail page. Sees worker tasks, shared tasks, AND employer tasks (laptop provisioning, SSO group setup) — the worker only sees worker + shared.

Next: Offboard →
  1. Workforce → click a worker row → detail page

    Open a worker detail.

    Pick any direct-sourced worker (the same one you onboarded in section 03 or 04, or any seed worker with status Onboarding). The detail page renders its standard accordion stack — Details, Engagements, Compliance, Performance, Schedule, Blocked sites, Logs.

  2. Spot the new Onboarding accordion.

    Slots in directly after Details, open by default when status is Onboarding or Offboarding. Hidden for agency-sourced Frontline workers (their lifecycle belongs to the agency) and for IC contractors (they have their own tracker on the IC detail). Subtitle reads either Professional engagement · task catalog from Settings or Frontline direct sourcing · task catalog from Settings.

  3. Read the head row.

    Two cells: left shows X / Y complete · NN% in tabular numerals, right shows a one-line projection — ~3 days until shift-ready for Onboarding, Off-boarded for closed Offboarding. A progress bar follows, blue when Onboarding, yellow when Offboarding.

  4. Scan the three task groups.

    Tasks group by owner — Worker tasks first (what the worker is doing now), Employer tasks (laptop, SSO, badge), Shared tasks (orientation, kickoff). Each row shows the task name (struck through when done), a "Due in Nd" stamp, a connector tag, and an inline action button per state — Remind for stalled worker tasks, Retry for blocked tasks, Mark done for in-progress employer tasks.

    Try: Tap Remind on a worker task — toast confirms the reminder went to the worker. In production this would email the worker; here it's a prototype hook.

  5. Watch the accordion close on completion.

    Once the worker side completes every required task in section 05, the accordion's progress bar fills, status flips to Compliant on the worker hero, and the accordion auto-collapses to a single check line. The worker is shift-ready.

07 · Manager · offboard a worker

Same Remove panel that ships today — scope enum lifted from shift to engagement / worker.

The existing RemoveWorkerPanel already covers shift-scoped removal (this shift / upcoming / all / pick specific). We extend the scope enum with two new values — engagement for Pro, worker for direct Frontline — and the panel branches into three new step bodies: effective date + reason picker, offboarding task acknowledgement + final-pay note, review. No new file, no new model, same panel host.

↩
Role

Manager · ending an engagement

Initiates offboarding from the row menu or the detail-header overflow. For Pro that's the engagement ending (term ran out, role consolidated, voluntary resignation). For direct Frontline that's the assignment ending (seasonal close, end of assignment, performance / for cause).

Next: Worker exit →
  1. Workforce → row menu → Offboard worker

    Initiate from the row menu.

    Direct-sourced workers (Pro engagement OR Internal / Float / Per-diem / Alumni pool) now show Offboard worker in the row menu instead of Remove worker. Same overflow on the detail-page header. Agency-sourced Frontline workers keep the original "Remove worker" / "Remove from booking" wording — their offboarding belongs to the agency.

    Default scope: the menu action pre-sets engagement for Pro workers and worker for direct Frontline. The panel opens already in the right shape — Manager doesn't have to pick scope.

  2. Fill effective date + reason.

    Step 1 opens with the worker context strip (avatar, name, supplier chip, role), then two date fields side-by-side — Effective date (defaults to 14 days from today) and Last day worked. Below them: the Reason dropdown — twelve options shared across shift + lifecycle scopes:

    • End of engagement term — engagement reached its scheduled end
    • Voluntary resignation — worker resigned, with or without notice
    • End of assignment · Seasonal close — Frontline specifics
    • Reorganization or role elimination — Pro reorg
    • Performance or conduct · For cause — gates the do-not-rehire flag on step 2
    • Role no longer needed · Other

  3. Pick last-shift handling (Frontline only).

    For Frontline direct scope, a third section appears — Last-shift handling with three options:

    • Work through-date — honor every shift booked through the effective date
    • End after current shift — finish the shift in progress, cancel everything after
    • End immediately — cancel all future shifts the moment you confirm

    Pro engagements skip this — they don't have shifts.

  4. Tour Step 2 · Tasks & final pay.

    The panel reads the offboarding catalog from window.getOffboardingTasks(kind) — the same one the admin authored in section 02 — and shows it as an acknowledgement list. Each task is on by default; uncheck to skip with audit-logged justification. Below: a Final-pay note textarea (routed to Dayforce native payroll) and the policy-gated Do-not-rehire flag.

    DNR gating: the do-not-rehire flag is grayed out unless the reason on step 1 is Performance / For cause / No-show. Voluntary or end-of-term separations can't set this flag — the policy is enforced in the panel, not just on submit.

  5. Review and confirm.

    Step 3 renders a different headline depending on scope — End <worker>'s engagement? for Pro, Offboard <worker>? for Frontline. Eight rows in the InfoGrid summarize scope, dates, reason, last-shift handling, DNR flag, final-pay note. Tap the acknowledgement checkbox at the bottom; the footer button reads End engagement or Offboard worker depending on the scope.

    State machine: on confirm, the worker transitions from Compliant to Offboarding immediately. After every required offboarding task completes, status flips to Inactive (or Terminated if DNR was set).
08 · Worker · self-serve offboarding

Same On-board tab repurposes when status flips to Offboarding.

W
Role

Worker · winding down

Sees a yellow-toned hero replacing the blue onboarding hero, and a shorter task list — acknowledge final pay, confirm asset returns received, optionally complete the exit interview.

Next: Closeout →
  1. Worker mobile · bottom bar

    Spot the Off-board tab.

    Same tab slot as On-board; label flips to Off-board and the badge count restarts. The tab is the first item in the bar so the worker can't miss it.

  2. Read the yellow hero.

    Hero block flips from the blue Onboarding gradient to a yellow Offboarding gradient. Headline reads Let's wrap up, <first name>. Sub-line: A few last steps before your final paycheck closes. The app will sign you out once everything is acknowledged.

  3. Walk the exit task list.

    Same card shape, shorter list — worker-owned offboarding tasks only. For Pro: knowledge transfer write-up, exit interview link. For Frontline: asset return acknowledgement (badge, uniform, locker key), final-paycheck acknowledgement, P45 download (UK) or final pay statement (US).

    📦Asset return
    Worker confirms each asset returned. Manager sees the matching employer task auto-close on the tracker accordion.
    💰Final pay ack
    Worker reviews the final-pay statement (total amount + breakdown). Routed to Dayforce native payroll on acknowledgement.
    🎤Exit interview
    Optional. Opens an HR-led 30-minute scheduling flow. Skippable; the audit log notes who declined.

  4. Watch the app sign the worker out.

    When every required exit task is acknowledged, status flips to Inactive (voluntary / end-of-term) or Terminated (DNR-flagged offboardings). The Off-board tab disappears, the mobile shell shows a brief farewell screen, the worker is signed out.

09 · Lifecycle complete

Six states, three transitions, one audit trail.

The new state machine extends the existing three-state worker enum (Onboarding / Compliant / Expired) with three new states for the back of the lifecycle — Offboarding · Inactive · Terminated. The existing chip palette paints them automatically; no new chip primitive.

Onboarding → Compliant → Offboarding → Inactive or Terminated Expired branches off Compliant when credentials lapse; reachable from Onboarding too if the worker times out on a required task.

  1. Read the audit log.

    The existing Logs accordion on the worker detail picks up every lifecycle transition automatically — actor + timestamp + payload diff for every task assigned / started / completed / skipped / failed, every status flip, every panel submit. Same writeAudit helper SOW Milestones, Contractor Agreements, and Frontline shift removals already use. The accordion gains a new Lifecycle event-type chip on the filter row.

  2. Check the webhook trail.

    Five new events on the webhook catalog — worker.onboarding.started, worker.onboarded, worker.offboarding.started, worker.offboarded, worker.terminated. Each carries worker id, worker type, triggering actor, reason code, and the lifecycle task transition payload. Same catalog the existing contractor.* events sit on.

  3. Reset the demo.

    Reload the page. localStorage holds the feature-flag state + the lifecycle program catalogs; everything else (worker statuses, panel state) resets to seed.

    To clear all customization: open Settings → Configuration → Pro lifecycle (or Frontline lifecycle) and tap Restore defaults. The Admin's hand-authored task tweaks reset to the seed catalogs.
End of walkthrough

That's the full worker lifecycle inside Flex Work — Admin configures two program cards, Manager onboards under either of two competitor-shaped flows (Fieldglass + VNDLY for Pro, Joined Up for Frontline), Worker self-serves through a single mobile tab that repurposes for exit, Manager tracks progress on a single accordion that picks up every task transition. The 31 parity tasks behind this flow live in On-/Offboarding parity tasks; the architectural mapping shows every task hangs off an existing surface — one new top-level file in the IA (the lifecycle program-card host, mirroring SOW + IC).
