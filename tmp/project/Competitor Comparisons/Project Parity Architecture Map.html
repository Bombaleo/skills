Architectural principles

Eight rules every Project-parity decision below honours. Break one and the system stops scaling cleanly.

Principle 01

Single domain module owns the data

Lifecycle states, transitions, NTE helpers, phase/task hierarchy, amendment shape, audit-entry shape, document shape, storage I/O — every primitive lives in pages/project-engagement.jsx. Adding a field is a one-file edit.

Principle 02

Views split by role, then by tab

Today pages/project-engagement-views.jsx holds every component. As surface grows past ~1500 lines, split per role: -admin.jsx, -manager.jsx, -agency.jsx, -worker.jsx. Detail tabs (Burn, Roster, Amendments…) stay as siblings inside -manager.jsx.

Principle 03

Host pages stay thin

Settings, new-requisition, requisitions, worker-mobile know nothing about Project except how to mount the component (window.ProjectAdminPanel, window.ProjectIntakeFields, …) behind the engProject flag. Project logic never bleeds into host files.

Principle 04

Variant registry routes detail pages

Project registers with window.VariantRegistry against channel "Project" + id prefix "PRJ-". The unified detail router does the rest. Adding a sub-type (e.g. capex-funded Project) is a new manifest, not a new route.

Principle 05

Per-org storage namespace

Everything keyed on flexwork.projects.{orgId} and flexwork.projectConfig.{orgId}. Org switches re-read; never re-fetch globally. New shapes (status reports, compliance acks) extend in-place or sit beside under the same prefix.

Principle 06

Events for cross-module sync

projects:change + projectconfig:change + featureflags:change + industry:change. Anything that wants to react to Project state subscribes — never reaches across module boundaries directly.

Principle 07

Cross-cutting features extend existing modules

Compliance scoped to a project lives in pages/compliance.jsx; resource forecasting lives in pages/analytics.jsx; FX-as-of-date lives in a financial-foundation module. Project module references those, never re-implements.

Principle 08

Two-party flows share components

Where manager + supplier touch the same object (closeout, amendments, roster, resource swap), one component takes a tenantRole prop and inverts. No duplicated Manager and Agency forks of the same modal.

The Project surface in the codebase today

Two new modules plus a thin layer of mount points across existing host pages. Everything below is gated by the engProject feature flag — off and the entire surface disappears.

pages/project-engagement.jsx
Core · domain + storage
States, transitions, helpers, per-org storage, seed data, variant registration, REQUISITIONS injection. Every Project data shape originates here. Exports primitives on window.PSProjects.
pages/project-engagement-views.jsx
Core · React components
ProjectAdminPanel, ProjectIntakeFields, ProjectVariantBody (with eight tab components), AgencyProjectsTab, WorkerProjectScreen, plus the burn chart, progress, pill, modal, field-grid primitives. Exports on window.*.
styles-project-engagement.css
Core · styling
Every .prj-* and .wm-project-* class. Layered on top of tokens.css — uses Everest tokens for colour, type, spacing. Standalone so it can be loaded once at index.html top.
pages/settings-config.jsx
Host · admin mount point
Renders <window.ProjectAdminPanel /> when engProject is on, behind a "Project settings" rail item. Otherwise null.
pages/new-requisition.jsx
Host · intake mount point
Renders <window.ProjectIntakeFields /> after the engagement-type picker when engType === "Project". Captures budget / NTE / currency / external ID into local state, committed to a Project record on Review.
pages/requisitions.jsx
Host · list mount point
Reads window.REQUISITIONS — which the core module post-load injects Project rows into. Also renders <window.AgencyProjectsTab /> above the list on agency tenants.
pages/requisition-engagement-detail.jsx
Host · variant router
Dispatches PRJ-* ids to ProjectVariantBody via the registry. No Project-specific code — the manifest carries everything.
pages/worker-mobile.jsx
Host · worker mount point
Adds a "Projects" tile to the More screen when engProject is on; opens <window.WorkerProjectScreen /> in a panel.
